## For Privatpersonen

<SoundEffect name="keyboard-typing" />

Du behøver ikke være <Voice text="I.T.">IT</Voice>-ekspert for at beskytte dig selv. Her er de vigtigste skridt:

### Uge 1: Adgangskoder

| Handling | Hvorfor |
|----------|---------|
| **Installér en password manager** | Huske 100 unikke passwords er umuligt for mennesker |
| **Generér unikke passwords til alt** | Ét kompromitteret password skal ikke åbne alt |
| **Skift passwords til kritiske konti** | Bank, email, sociale medier |

**Anbefalet:** <Voice text="Bit Warden">Bitwarden</Voice> (gratis), <Voice text="1 Password">1Password</Voice> eller <Voice text="Dash Lane">Dashlane</Voice>.

<Pause duration={1} />

### Uge 2: <Voice text="To-faktor-autentifikation">To-faktor</Voice>

| Handling | Hvorfor |
|----------|---------|
| **Aktivér <Voice text="M.F.A.">MFA</Voice> på alle kritiske konti** | Stjålet password er ikke nok uden din telefon |
| **Brug authenticator-app** | Bedre end <Voice text="S.M.S.">SMS</Voice>, som kan kapres |
| **Gem backup-koder sikkert** | Hvis du mister din telefon |

**Anbefalet:** <Voice text="Microsoft Authenticator">Microsoft Authenticator</Voice>, <Voice text="Google Authenticator">Google Authenticator</Voice> eller <Voice text="Authy">Authy</Voice>.

### Uge 3: Opdateringer

| Handling | Hvorfor |
|----------|---------|
| **Slå automatiske opdateringer til** | Sikkerhedshuller lukkes automatisk |
| **Tjek telefon og computer** | Ældre enheder får ikke opdateringer |
| **Opdatér browseren** | Dit vindue til internettet skal være sikkert |

### Uge 4: Backup

| Handling | Hvorfor |
|----------|---------|
| **3-2-1 backup-reglen** | 3 kopier, 2 medietyper, 1 offsite |
| **Test at backups virker** | En backup du ikke kan gendanne er værdiløs |
| **Kryptér følsomme backups** | Stjålne backups skal stadig være ulæselige |

## For Udvikleren

<Music track="ambient-tech" action="fade-in" />

Du bygger softwaren, andre stoler på. Dit ansvar er større.

### Dag 1: Kend dine afhængigheder

```bash
# Node.js
npm audit

# Python
pip-audit

# .NET
dotnet list package --vulnerable
```

Kør dette dagligt. Automatisér det i din <Voice text="C.I.">CI</Voice>-pipeline.

<Pause duration={1} />

### Dag 2: Generér din <Voice text="S-BOM">SBOM</Voice>

```bash
# Node.js
npx cyclonedx-npm --output-format json > sbom.json

# Python
pip install cyclonedx-bom
cyclonedx-py -o sbom.json

# Multi-platform
syft . -o cyclonedx-json > sbom.json
```

<View>Gem SBOM'en som en del af din release-proces.</View>

### Dag 3: Sikker kode

| Praksis | Implementering |
|---------|----------------|
| **Secrets management** | Aldrig credentials i kode — brug <Voice text="environment variables">environment variables</Voice> eller vaults |
| **Input validation** | Stol aldrig på brugerinput |
| **Dependency pinning** | Lås versioner for reproducerbare builds |
| **Code review** | Mindst to øjne på sikkerhedskritisk kode |

<Music track="ambient-tech" action="fade-out" />

### Løbende: Bliv opdateret

- Følg sikkerhedsblogs for dine teknologier
- Abonnér på <Voice text="C.V.E.">CVE</Voice>-feeds for dine afhængigheder
- Deltag i sikkerhedscommunitiet
