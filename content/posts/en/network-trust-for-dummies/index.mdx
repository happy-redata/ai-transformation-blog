### How does VPN work?
1. You start an encrypted tunnel to the company network
2. Your traffic goes through this tunnel
3. The firewall sees you as an "internal" user
4. You get access to everything internal

### The Problem
<SoundEffect name="buzzer-wrong" />

Once you're "inside" via VPN, the system trusts you. It's the classic "castle" model:

- If your laptop is compromised, the attacker now has **full access**
- Lateral movement (hopping from system to system) is easy
- VPN credentials become targets for phishing

<Pause duration={1} />

> **"VPN solves the problem of getting in. It doesn't solve the problem of what happens once someone is inside."**

### Zero Trust Principles

| Principle | Meaning |
|-----------|---------|
| **Verify explicitly** | Authenticate and authorize based on all available data points |
| **Least privilege** | Only give access to what's necessary â€” nothing more |
| **Assume breach** | Design as if the network is already compromised |

### Example: Accessing a file

**Castle model (VPN):**
1. Log in via VPN âœ…
2. Access to all files on the server âœ…

**Zero Trust:**
1. Is the user who they claim to be? (MFA) âœ…
2. Is the device compliant? (Updated, encrypted) âœ…
3. Does the user have specific access to this file? âœ…
4. Is it normal for the user to access this file at 3 AM from Russia? âŒ
