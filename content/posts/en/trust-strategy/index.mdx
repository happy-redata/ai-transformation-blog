## For the Individual

<SoundEffect name="keyboard-typing" />

You don't have to be an <Voice text="I.T.">IT</Voice> expert to protect yourself. Here are the most important steps:

### Week 1: Passwords

| Action | Why |
|----------|---------|
| **Install a password manager** | Remembering 100 unique passwords is impossible for humans |
| **Generate unique passwords for everything** | One compromised password should not open everything |
| **Change passwords for critical accounts** | Bank, email, social media |

**Recommended:** <Voice text="Bit Warden">Bitwarden</Voice> (free), <Voice text="1 Password">1Password</Voice>, or <Voice text="Dash Lane">Dashlane</Voice>.

<Pause duration={1} />

### Week 2: <Voice text="Two factor authentication">Two-factor</Voice>

| Action | Why |
|----------|---------|
| **Activate <Voice text="M.F.A.">MFA</Voice> on all critical accounts** | A stolen password is not enough without your phone |
| **Use an authenticator app** | Better than <Voice text="S.M.S.">SMS</Voice>, which can be hijacked |
| **Save backup codes securely** | In case you lose your phone |

**Recommended:** <Voice text="Microsoft Authenticator">Microsoft Authenticator</Voice>, <Voice text="Google Authenticator">Google Authenticator</Voice>, or <Voice text="Authy">Authy</Voice>.

### Week 3: Updates

| Action | Why |
|----------|---------|
| **Turn on automatic updates** | Security holes are closed automatically |
| **Check phone and computer** | Older devices don't get updates |
| **Update the browser** | Your window to the internet must be secure |

### Week 4: Backup

| Action | Why |
|----------|---------|
| **The 3-2-1 backup rule** | 3 copies, 2 media types, 1 offsite |
| **Test that backups work** | A backup you cannot restore is worthless |
| **Encrypt sensitive backups** | Stolen backups should still be unreadable |

## For the Developer

<Music track="ambient-tech" action="fade-in" />

You build the software that others trust. Your responsibility is greater.

### Day 1: Know your dependencies

```bash
# Node.js
npm audit

# Python
pip-audit

# .NET
dotnet list package --vulnerable
```

Run this daily. Automate it in your <Voice text="C.I.">CI</Voice> pipeline.

<Pause duration={1} />

### Day 2: Generate your <Voice text="S-BOM">SBOM</Voice>

```bash
# Node.js
npx cyclonedx-npm --output-format json > sbom.json

# Python
pip install cyclonedx-bom
cyclonedx-py -o sbom.json

# Multi-platform
syft . -o cyclonedx-json > sbom.json
```

<View>Save the SBOM as part of your release process.</View>

### Day 3: Secure code

| Practice | Implementation |
|---------|----------------|
| **Secrets management** | Never credentials in code — use <Voice text="environment variables">environment variables</Voice> or vaults |
| **Input validation** | Never trust user input |
| **Dependency pinning** | Lock versions for reproducible builds |
| **Code review** | At least two sets of eyes on security-critical code |

<Music track="ambient-tech" action="fade-out" />

### Ongoing: Stay updated

- Follow security blogs for your technologies
- Subscribe to <Voice text="C.V.E.">CVE</Voice> feeds for your dependencies
- Participate in the security community

## For the Leader

<Speaker voice="narrator">
You set the direction. Your support — or lack thereof — determines whether security is prioritized.
</Speaker>

### Strategic priorities

| Priority | Action | Investment |
|-----------|----------|-------------|
| **1. Identity** | <Voice text="M.F.A.">MFA</Voice> and <Voice text="Single Sign-On">SSO</Voice> for all | Low-medium |
| **2. Visibility** | Logging and monitoring | Medium |
| **3. Endpoint** | Modern endpoint protection | Medium |
| **4. Network** | Segmentation and <Voice text="Zero Trust">Zero Trust</Voice> Architecture | High |
| **5. Culture** | Awareness training | Low |
